介绍：
	基于操作系统编程实验和操作系统课设共用。
	经过重构之后代码可读性已经比较强了。但是健壮性依然不够。
	上层代码几乎都是调用。而底层存在逻辑缺陷。
问题：
	1.上层缺少一个查找文件的函数，导致操作层代码重复
	2.inode虽然可以有8个块，但是实际使用只有一个。文件超出1024B时会异常
	3.虽然创建文件时要输入大小和mode，但是后台并未使用。文件属性需要后期整理
	4.xmlUtils因为写的比较早，只求实用，因此扩展性极差。需要大规模重构。
	5.由于xmlUtls的限制，实际没有创建用户功能
	6.输入密码应改为输入不回显。目录显示应显示用户和路径
